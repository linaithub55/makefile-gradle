## 1. Автоматизированная сборка C++ (Makefile)

Создан `Makefile`, автоматизирующий сборку C++ программы. Для компилятора использована мета-переменная `CXX=g++`.

**Содержимое `Makefile`:**
```makefile
CXX = g++
CXXFLAGS = -Wall -Wextra -std=c++17

# 1. Сборка и получение исполняемого файла
all: build
build: ChessMath.o
	$(CXX) $(CXXFLAGS) main.cpp ChessMath.o -o a.out

# Компиляция объекта библиотеки
ChessMath.o: ChessMath.cpp ChessMath.h
	$(CXX) $(CXXFLAGS) -c ChessMath.cpp -o ChessMath.o

# 2. Сборка с отладочными символами (-g)
debug: CXXFLAGS += -g
debug: build_debug

build_debug: ChessMath_debug.o
	$(CXX) $(CXXFLAGS) main.cpp ChessMath.o -o a.out_debug

ChessMath_debug.o: ChessMath.cpp ChessMath.h
	$(CXX) $(CXXFLAGS) -c ChessMath.cpp -o ChessMath.o

# 3. Очистка директории
clean:
	rm -f *.o a.out a.out_debug
```

**Выполнение целей:**
1. Команда `make` (или `make build`) — компилирует исходный код в исполняемый файл `a.out`.
2. Команда `make debug` — собирает программу с отладочными символами (опция `-g`), создавая `a.out_debug`.
3. Команда `make clean` — очищает директорию от всех промежуточных объектных файлов (`.o`) и бинарников.

## 2. Переход на Java и использование Gradle

Основная логика шахматных угроз переписана на Java. Проект настроен с использованием системы сборки **Gradle** (создан конфигурационный файл `build.gradle`).

### Объяснение задач Gradle (`gradle tasks`)
Если ввести в консоль `gradle tasks`, система выведет доступные цели сборки. Основные из них:
- `gradle build` — основная задача сборки. Компилирует Java-код, затем компилирует тесты, выполняет их прогон и пакует итоговый результат в архив.
- `gradle test` — запускает только этап Unit-тестирования и генерирует отчет.
- `gradle clean` — очищает проект, полностью удаляя папку `build/` со всеми временными файлами (аналог `make clean`).
- `gradle run` — компилирует и мгновенно запускает программу (используется плагин `application`).

### Путь к исполняемому файлу и запуск через консоль
После выполнения `gradle build`, скомпилированные байт-коды (файлы `.class`) можно найти по пути:
```text
build/classes/java/main/chess/
```

Запуск программы осуществляется через консоль. Находясь в корневой папке Java-проекта, нужно указать JVM путь к скомпилированным классам через параметр `-cp` (ClassPath) и передать аргументы командной строки:
```bash
$ java -cp build/classes/java/main chess.Main 4 4 4 8 1 1
Координаты загружены из аргументов командной строки.
Угроза от обеих фигур
```

## 3. Unit-тестирование (JUnit 4)

В соответствии с заданием, система тестирования реализована на Java. Написан класс `ChessMathTest.java`, использующий встроенные функции `Assert.assertEquals`.

**Пример теста (перекрытие ладьи слоном):**
```java
@Test
public void testSafetyWithBlockRook() {
    // Король(1,1), Ладья(1,8), Слон(1,4)
    // Ладья угрожает по вертикали, но слон стоит строго между ними.
    int result = ChessMath.checkThreats(1, 1, 1, 8, 1, 4);
    Assert.assertEquals("Король должен быть в безопасности из-за перекрытия ладьи (0)", 0, result);
}
```

**Отчет о тестировании:**
Gradle формирует красивые и подробные HTML-отчеты о пройденных тестах. После прогона `gradle test`, результат в виде веб-страницы сохраняется по пути:
```text
build/reports/tests/test/index.html
```
Все 5 написанных тестов успешно пройдены (проверка всех угроз и блокировок линий/диагоналей).
